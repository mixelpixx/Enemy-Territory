## Task 1: Real JPEG decode in the DLL

**Files:** `src/renderer2/bridge/tr2_jpeg_compat.c|h` (new — the mem-src shim), `cmake/Renderer2.cmake`, delete `bridge/tr2_jpg_stub.c` from target+tree

- [ ] **Step 1:** Read the vendored `etl/common/tr_image_jpg.c`: enumerate exactly which libjpeg-8 APIs it uses beyond jpeg-6b (expected: `jpeg_mem_src`, possibly `jpeg_mem_dest` in the save path, `JERR_*` usage). Our bundled `src/jpeg-6` (target `etrm_jpeg`) provides the rest.
- [ ] **Step 2:** Implement the classic mem-source shim (`tr2_jpeg_compat.c`): a `jpeg_source_mgr` over a memory buffer providing `init_source/fill_input_buffer/skip_input_data/resync_to_restart/term_source` + a `jpeg_mem_src(j_decompress_ptr, const unsigned char*, unsigned long)` wrapper (the well-known ~70-line adapter ioquake3 used over jpeg-6b). If the save path needs `jpeg_mem_dest`, same pattern with a growable dest mgr (or stub the SAVE path only — screenshots can stay TGA — documented). Header exposes the prototypes; compile order must see jpeg-6b's `jpeglib.h` (include dirs from `src/jpeg-6`).
- [ ] **Step 3:** CMake: include `etl/common/tr_image_jpg.c` in the target (remove from REMOVE_ITEM), add the shim TU, link `etrm_jpeg`, add `src/jpeg-6` include dir FOR THOSE TUs (scoped — don't leak into the rest of the vendored tree). Delete `tr2_jpg_stub.c` (tree + target). WATCH: jpeg-6b's `jpeglib.h` config (`jconfig.h`) is in our tree; their tr_image_jpg.c may set `JPEG_INTERNALS` or expect `boolean` differences (jpeg-6b defines `boolean` as int — MSVC + their headers may clash with windows.h's `boolean` — known issue; mechanical guards allowed, logged).
- [ ] **Step 4:** Verify: build; then a focused launch loading a jpg-textured asset — simplest objective check: gl2 menu launch, then grep the log for the previous "jpg placeholder" logged-once warning = should be GONE; plus Task 2's world load is the real proof. If their save path was stubbed, note it. ctest 3/3. Commit.
- [ ] Fallback (only if jpeg-6b proves intractable — escalate first): FetchContent libjpeg-turbo (pinned), link the DLL against it instead; THIRDPARTY row.
